1
Installer les outils de base
Node.js, Git, VS Code — à faire une seule fois par poste

🟢 Node.js 20+

nodejs.org/en/download

Choisir la version LTS. Inclut npm. Vérifier : node -v

🐙 Git

git-scm.com/downloads

Inclus sur Mac/Linux. Sur Windows : Git for Windows. Vérifier : git --version

💙 VS Code

code.visualstudio.com

Éditeur principal. Extensions recommandées : ESLint, Tailwind CSS IntelliSense, GitLens.

⚙️ Configurer Git identité
git config --global user.name "Votre Nom"
git config --global user.email "votre@email.com"
🔑 Connecter GitHub via SSH (recommandé)
# Générer une clé SSH
ssh-keygen -t ed25519 -C "votre@email.com"

# Afficher la clé publique à copier dans GitHub
cat ~/.ssh/id_ed25519.pub

Ensuite dans GitHub → Settings → SSH and GPG keys → New SSH key → coller la clé.

2
Configurer Claude Code
L'assistant IA qui connaît déjà tout le contexte du projet
📦 Installer Claude Code
npm install -g @anthropic-ai/claude-code
🔐 Authentifier Claude Code
claude

La première fois, Claude Code ouvre le navigateur pour se connecter à votre compte Anthropic (claude.ai). Suivez les instructions à l'écran.

💡 Extension VS Code Claude Code

Dans VS Code → Extensions (Ctrl+Shift+X) → chercher Claude Code → Installer.
Ou depuis la palette de commandes : Ctrl+Shift+P → Claude Code: Open.

💡 Claude Code lit automatiquement le fichier CLAUDE.md à la racine du projet et charge la mémoire persistante dans ~/.claude/projects/. Il retrouve tout le contexte de la session précédente automatiquement.
3
Cloner le dépôt
Récupérer le code source depuis GitHub
📂 Cloner topsysapp_v2
# Choisir un dossier de travail, ex: Documents/projets
cd ~/Documents

# Cloner le dépôt de travail (pas TopsysExplorer original)
git clone https://github.com/MyData-Folk/topsysapp_v2.git

cd topsysapp_v2
📦 Installer les dépendances
npm install

Installe React 19, Vite 6, Supabase, Tailwind CSS 4, Recharts, pdfjs-dist, etc.

🔍 Vérifier l'état du dépôt
git log --oneline -5   # Voir les derniers commits
git status             # Vérifier qu'on est sur main propre
git remote -v          # Vérifier les remotes
⚠️ Le dépôt original est TopsysExplorer (lecture seule). Le dépôt de travail actif est topsysapp_v2. Toujours travailler sur ce dernier.

5
Lancer le serveur de développement
Hot-reload sur http://localhost:3000
🚀 Démarrer Vite
npm run dev

L'app s'ouvre sur http://localhost:3000 (ou 3001 si 3000 est occupé).

🔧 Autres commandes utiles
npm run lint      # TypeScript type check (tsc --noEmit)
npm run build     # Build production → dossier dist/
npm run preview   # Tester le build de production localement
✅ Si l'écran de connexion Topsys Explorer apparaît → tout est configuré correctement.
6
Workflow GitHub
Branches, commits, push, pull requests
🌿 Créer une branche de fonctionnalité
# Toujours partir de main à jour
git checkout main
git pull

# Créer une nouvelle branche
git checkout -b feat/ma-nouvelle-feature
💾 Committer son travail
git add src/components/MonComposant.tsx
git commit -m "feat(composant): description de la fonctionnalité"

Convention de commits : feat: nouvelle fonctionnalité, fix: correction, chore: maintenance.

⬆️ Pousser et créer une PR
git push topsysapp_v2 feat/ma-nouvelle-feature

# Créer la PR via GitHub CLI
gh pr create --repo MyData-Folk/topsysapp_v2 \
  --base main \
  --title "feat: description" \
  --body "Description des changements"
🔀 Merger sur main
# Une fois la PR validée, merger localement
git checkout main
git merge feat/ma-nouvelle-feature
git push topsysapp_v2 main

8
Reprendre une session Claude Code
Comment briefer Claude pour qu'il retrouve tout le contexte
📂 Ouvrir le projet dans VS Code
cd ~/Documents/topsysapp_v2
code .

VS Code avec l'extension Claude Code charge automatiquement le contexte du projet.

💬 Phrase de reprise recommandée

Dans Claude Code, commencer par :

"Consulte ta mémoire et analyse l'application. Nous travaillons sur TopsysExplorer v2 (repo topsysapp_v2). Voici ce que je veux faire aujourd'hui : [description]"
🧠 Ce que Claude retrouve automatiquement
  • Infos Coolify (URL, token, UUID app) via la mémoire persistante
  • Infos Supabase (URL, clé JWT) via la mémoire persistante
  • Architecture du projet via CLAUDE.md
  • Contexte code via les fichiers source du projet
  • Historique des décisions via git log
📋 Fichier de contexte à donner à Claude

Voir NEXT_SESSION.md à la racine du projet — il contient l'état exact du projet, les fonctionnalités implémentées, les bugs connus et les prochaines étapes.

✅ Claude Code charge la mémoire persistante depuis ~/.claude/projects/ — les tokens, URLs et décisions architecturales sont déjà là, pas besoin de les re-saisir.
